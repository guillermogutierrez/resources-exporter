Typical Liferay infrastructure diagram

If the information needs to be accessed from different geographical regions, and trying to avoid latency issues even when CDN are in place, this environment can be replicated partial o totally in every single region.

Liferay servers

  • Staging environment: Two nodes for editing and publishing content
  • Live environment: Two / Three nodes for live content

Database 

 At least one database per environment: staging and live

HttpServers

 It's a good practice to have a HTTP server layer on front of the Liferay servers

Document Storage

Depending on the amount of documents to deal with considerations about platform will be completely different

CDN

As the services will be accessed from two different continents using a CDN to deliver all static content will help to reduce latency and downloading times. 

Search

At least one dedicated elastic search server

One server with kibana software or any other tool to help dealing with the search information.
